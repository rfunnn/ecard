2. Pre-Deployment Decision

Initially attempted to pull a pre-built Docker image from GitHub Container Registry (GHCR) but received an unauthorized error. The GitHub Actions CI/CD pipeline had not successfully built and pushed the image.

Decision: Build the Docker images directly on the VPS from source code instead of pulling from GHCR. This removes the dependency on GitHub Actions for the initial deployment.

4. Dockerfile Fixes

Three issues in the Dockerfile were discovered and fixed during the deployment process.

Fix 1 — Remove deprecated --skip-generate flag
Issue: Prisma 7 removed the --skip-generate flag from prisma db push. The migrator container kept restarting with "unknown or unexpected option".
Fix: Removed the flag from the ENTRYPOINT in the migrator stage.
# Before
ENTRYPOINT ["npx", "prisma", "db", "push", "--skip-generate"]

# After
ENTRYPOINT ["npx", "prisma", "db", "push"]
Fix 2 — Move migrations into the app container

Removed the separate migrate service from docker-compose.yml and instead run prisma db push inside the app container at startup via an entrypoint script.

Created entrypoint.sh at the project root:

#!/bin/sh
set -e
node node_modules/prisma/build/index.js db push --accept-data-loss
exec node server.js

Updated the runner stage in the Dockerfile:

# Copy full node_modules so Prisma CLI has its complete dependency tree
COPY --from=builder --chown=nextjs:nodejs /app/node_modules ./node_modules
COPY prisma/schema.prisma  ./prisma/schema.prisma
COPY prisma.config.ts      ./
COPY package.json          ./
COPY --chmod=755 entrypoint.sh ./

CMD ["./entrypoint.sh"]

Updated docker-compose.yml — app service depends_on changed from migrate to just db and storage.

Fix 3 — Prisma WASM file path issue
Issue: Copying only node_modules/.bin/prisma caused "ENOENT prisma_schema_build_bg.wasm" because Docker dereferences the symlink, making __dirname wrong — the CLI then looks for WASM files in .bin/ instead of prisma/build/.
Fix: Call the prisma CLI directly via node node_modules/prisma/build/index.js in entrypoint.sh (no symlink involved), and copy the full node_modules from the builder stage to ensure all transitive dependencies (effect, @prisma/config, etc.) are available.

6. Domain and SSL Setup

Step 1 — Add DNS A record at GoDaddy

GoDaddy → My Products → ekadku.com → DNS → Manage DNS

TypeNameValueTTL
A@143.198.94.229600

Note: The existing www CNAME was left as-is (GoDaddy prevents adding a conflicting www A record).

Verify DNS propagation from VPS:

nslookup ekadku.com
# Expected: Address: 143.198.94.229
Step 2 — Connect Nginx Proxy Manager to the app network

NPM runs in its own Docker network. It needs to join the app's network to forward traffic to ekadku-app-1.

docker network connect ekadku_default npm

Verify NPM can reach the app:

docker exec npm curl -I http://ekadku-app-1:3000
# Expected: HTTP/1.1 200 OK
Step 3 — Configure Proxy Host in NPM UI

Open http://143.198.94.229:81 → Proxy Hosts → Add Proxy Host
